# DEMO.md：LPK Talent Platform 10分デモ台本

> BUILD_PLAN「ゴール」1〜5 を10分で通す台本。ローカルでも Azure 公開URLでも同じ手順で動く。
> 公開URLの構築・seed 投入は infra/terraform/README.md を参照。

## 0. 前提と起動

### 動作環境
- デモ標準機は Android Chrome（BUILD_PLAN の地雷：モバイル Safari のマイク挙動が不安定なため固定）。
- 開発機の PC Chrome でも可。いずれもマイク権限の許可が必要。
- 音声モードはブラウザの `MediaRecorder`（WebM/Opus）とマイクを使う。マイクは https か `localhost` でのみ許可される点に注意。

### プロバイダモード
音声モードは STT・LLM・TTS の3系統に依存する。既定の stub なら資格情報なしで通しで動く。ローカルは `backend/.env`、公開URLは terraform.tfvars（または GitHub Variables）で切り替える。

| 設定 | stub（既定） | 実プロバイダ |
|---|---|---|
| `provider_mode` | STT は定型の日本語回答を返す。録音内容は不問。ffmpeg 不要 | `azure`：Azure AI Speech で実音声を ja-JP 文字起こし・発音採点 |
| `llm_provider_mode` | 面接官の質問とルーブリック評価が決定的モック | `bedrock`：Bedrock の Claude が生成 |
| TTS（面接官の読み上げ） | 合成せず、ブラウザ内蔵の ja-JP 音声で読み上げ | `provider_mode=azure` のとき Azure Neural TTS の MP3（ja-JP-NanamiNeural） |

- 資格情報なしデモ（保険）：既定 stub のまま。録音（入口）と読み上げ（出口）は本物で、認識テキストと評価だけが定型になる。
- 品質デモ（推奨）：`provider_mode=azure` + `llm_provider_mode=bedrock`。azure のときのみサーバ側で ffmpeg の WAV(16kHz mono) 変換が走る（コンテナには同梱済み。ローカル backend 環境には ffmpeg を用意する）。

### 起動（ローカル）
別々のターミナルで順に実行する。

1. `make db`（Postgres を docker で起動）
2. `make migrate`
3. `make seed`（DB を初期化して架空データ投入。末尾にデモアカウントを表示する）
4. `make backend`（http://localhost:8001 ）
5. `make frontend` → ブラウザで http://localhost:5173 を開く

### 起動（Azure 公開URL）
- infra/terraform/README.md の bootstrap 済みであること。URL は `terraform output -raw frontend_url`。
- seed 再投入は `scripts/seed/seed_remote.sh <resource_group> <seed_job_name>`（デモアカウントも出力される）。
- ACA は scale-to-zero のため初回アクセスに数十秒かかる。**デモ5分前に一度アクセスして温めておく。**

## 1. 学生：発音練習（0:30〜2:30）

1. 学生（Siti）でログイン。ホームの「**Latihan pelafalan**」をタップ。
2. カード上部の「←」「→」で介護の文（職種バッジ「Perawatan lansia」）を選ぶ。日本語文 + ふりがな + インドネシア語訳が表示される。
3. 「**Ketuk untuk merekam**」をタップして読み上げ、「**Berhenti & nilai**」で採点（28秒で自動停止）。
4. 結果：「Skor keseluruhan」（総合）と Akurasi / Kelancaran / Kelengkapan。「**Skor per kata**」で単語チップが緑 / 黄 / 赤（Bagus / Cukup / Perlu latihan）に色分けされる。
5. 下の「**Kata yang perlu dilatih**」に弱点語が累積表示される（seed 仕込みの例：検温、手すり、賞味期限）。

話すポイント：単語レベルで弱点が可視化され、この同じデータが教師画面と Talent Passport にそのまま流れる。

## 2. 学生：音声面接ロールプレイ（2:30〜5:30）★本命

介護シナリオ「Wawancara: perawatan lansia」は候補者ターン5回で完了する（ゴール「介護、3〜5ターン」に対応）。

### 操作手順
1. ホーム → 「**Latihan wawancara AI**」。
2. 画面上部の「Mode:」トグルを **🎤 Suara** に切り替える。モードは新規セッションに適用されるので、シナリオを選ぶ前に切り替える。
3. シナリオ「Wawancara: perawatan lansia」をタップ。面接官の開幕質問（自己紹介）が自動で読み上げられる。聞き直しは各吹き出しの 🔊。
4. 「🎤 Bicara」をタップ → マイク許可 → 録音。録音中は赤い「⏹ Berhenti」と経過秒表示、45秒で自動停止。
5. 話し終えたら「⏹ Berhenti」。「Memproses...」の後、自分の回答（認識テキスト）が右に、面接官の次の質問が左に出て読み上げられる。
6. 計5回答すると、ルーブリック評価カードが出る（総合点 + 5軸：日本語力 / 内容の一貫性 / 敬語・礼儀 / 報連相 / 発話明瞭さ、インドネシア語の講評・アドバイス・模範解答）。
7. 一覧に戻ると履歴とスコア推移グラフに今の回が反映される。

### stub と実プロバイダの画面差（説明用）
- stub：録音内容にかかわらず回答は定型の日本語（例「シティと申します。インドネシアから参りました。」）。読み上げはブラウザ内蔵音声。疎通・操作感の確認向け。
- azure + bedrock：実際に話した日本語が認識テキストになり、質問と評価は Claude が生成。読み上げは Nanami の自然な声。品質デモ向け。

## 3. 教師ダッシュボード（5:30〜7:00）

1. 教師（田中美咲）でログイン。「**学生一覧**」が開く。

話すポイント：先生は録音を聞き直さなくても文字起こしと自動評価から確認でき、添削時間を削減できる（PoC KPI）。

## 4. Talent Passport（7:00〜8:00）

1. Siti の学生詳細下部「**Talent Passport**」セクション →「**Passportを生成**」。第n版と生成日が表示される。
2. 「**PDFダウンロード**」→ A4 日本語PDF（候補者紹介シート）。日本語が豆腐化していないこと、スコア・推移・チェックリストが1枚に収まっていることを見せる。

注意：PDF 生成は WeasyPrint 依存。コンテナ / 公開URLでは動くが、ローカル Windows 開発機では未導入だと 503 になる。ローカルデモで PDF を見せる場合は事前に動作確認しておく。

## 5. 企業向け共有リンクと候補者比較（8:00〜9:30）

1. 経営者（Hendra）でログイン → 上部「**企業向け共有リンクへ**」（`/admin/links`）→「**共有リンクを発行**」。有効期限は発行から30日。
2. 「リンクをコピー」し、**シークレットウィンドウで開く**。ログイン無しで見られることを強調（`/company/{token}`）。
3. 候補者比較テーブル：日本語レベル（推定）/ 発音平均 / 面接評価（直近）/ 面接回数 / 出席率 / チェックリスト / Passport 版。**列ヘッダーをクリックしてソート**（例：面接評価の降順 → 昇順トグル、▲▼表示）。
4. 氏名リンク → 個別 Passport ビュー（`/company/{token}/students/{id}`）。スコアカード・推移グラフ・生活態度・発話抜粋。**リスクフラグは企業ビューには出ない**方針であることを補足。「PDFダウンロード」もここから可能。
5. 時間があれば管理画面に戻り「**失効させる**」→ リンクが即 404 になることを見せる（閲覧ログ・失効管理も話す）。

補足：学生1名分の共有リンク（`/share/{token}`）は教師の学生詳細からも発行できる。

## 6. 経営者KPI（9:30〜10:00）

1. 「**経営者KPIダッシュボード**」（`/admin/kpi`）。主要カード（学生数 / リスク学生（赤）/ N4到達率 / 模試平均 / 出席率平均）、**PoC KPI カード5枚**（AI利用率 週3日以上 / 面接練習回数 / 模擬面接スコア改善 初回3回 vs 直近3回 / 模試スコア推移 / 添削キュー滞留）、週次トレンド（直近8週）、リスクフラグ学生リスト。
2. 締め：これらは BUILD_PLAN の KPI 表と同一定義で、全て events 等の実データから集計している。PoC 本番では学生30〜50名の実利用でこの画面がそのまま効果測定になる。

時間が余ったら：AI会話練習、デイリークイズ（N4 / JFT、非公式・試験対策用の表記あり）、模試モードを補足で見せる。

## 7. トラブルとフォールバック

### 現地トラブル対応
- マイク不許可：赤字「Mikrofon tidak bisa digunakan...」。ブラウザのマイク権限を許可して再度「🎤 Bicara」。
- 無音・認識失敗（azure 時）：「Suara tidak dikenali...」。もう一度はっきり話す。
- 読み上げが鳴らない：OS / ブラウザに ja-JP 音声が無い場合がある。🔊 で再試行するか、画面のテキストで代替する。
- 会場の音声が不安定：Mode を **Teks** に切り替えて同じ面接を実施する。
- 公開URLが遅い / 起きない：コールドスタートの可能性。1分待って再アクセス。復帰しなければローカル起動に切り替える。
- Azure Speech / Bedrock の障害：stub に切り替えて疎通デモに切り替える（ローカルは `.env` を戻して再起動、公開URLは Variables を stub にして再デプロイ。時間がかかるためローカル切替を優先）。
